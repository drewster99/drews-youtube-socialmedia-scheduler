Template syntax

{% raw %}

Variables — replaced with values from the surrounding context. "Empty" below means missing, empty, or whitespace-only.

  • {{name}} — substitute. An undefined name is an error (generation stops and names every undefined variable); a defined-but-empty value renders as nothing. Built-ins are always defined, so this only fires on typos or variables that don't exist in the render context.
  • {{name!}} — required content. Rendering fails when the value is missing or empty — use this for "never post without this link".
  • {{name??default text}} — substitute, or use the literal default text when the value is missing or empty. The default text is absolute — no variables, and no {{/}} braces at all, inside it. Use {{name??}} to render nothing.

Sections — include a whole block only when a variable has content.

  • {{#name}}…{{/name}} — the content between the tags renders only when name has a non-empty value. Anything inside a dropped section is skipped entirely: its variables aren't checked, its {{ai:}} blocks never call Claude, and its media directives attach nothing.
  • {{^name}}…{{/name}} — the inverse ("else"): renders only when name is missing or empty.
  • Sections nest, and can span multiple lines. Example:
    {{#episode_url}}Full episode: {{episode_url}}{{/episode_url}}{{^episode_url}}More at {{project_url}}{{/episode_url}}
  • An unclosed, mismatched, or malformed section tag is an error — nothing renders, so a broken tag can never leak into a post.

Built-in variables

  • {{title}}, {{description}}, {{description_short}} (≤150), {{description_medium}} (≤500), {{tags}}, {{hashtags}}, {{tier}}, {{user_message}}, {{thumbnail_path}}
  • {{transcript}} — full plain-text transcript (no timestamps). {{transcript_truncated}} — the same, capped (all render paths use one shared cap). {{transcript_srt}} — the stored SRT verbatim, timestamps intact. {{transcript_srt_truncated}} — SRT capped at a cue boundary. Mind the token bill when putting the untruncated forms inside {{ai:}} blocks on long episodes.
  • {{url}} — this item's URL (YouTube URL for YT-uploaded items, or whatever you set on a standalone item).
  • {{episode_url}} — the parent episode's URL when this item has a parent (e.g. a hook pointing at its primary episode).
  • {{parent_url}}, {{parent_title}}, {{parent_description}}, {{parent_tags}} — fields from the parent video when this item has one; empty strings otherwise. Wrap parent-dependent content in a section ({{#parent_url}}…{{/parent_url}}) so items without a parent don't get dangling labels.
  • {{parent_context_block}} — a ready-to-paste paragraph summarising the parent video (title, URL, description, tags), or empty when there's no parent. Designed to be opt-in via {{parent_context_block??}} inside prompt bodies.
  • {{project_url}} — the project's URL (auto-filled from the YouTube channel handle on YT projects, or set explicitly on GitHub / social-only projects).
  • {{max_chars}} — this slot's "Max characters" value, as a number. Handy inside an {{ai:}} block so the model knows the limit, e.g. {{ai: Write a post under {{max_chars}} chars about {{title}}}}. (Defined in post generation and the preview below; it doesn't exist in prompt-template bodies, where a bare reference is an error.)

Custom variables — define your own at three scopes; later wins on collision.

  • Global (lowest priority) — install-wide. Set under Settings. Useful for things like a default sign-off.
  • Project — overrides global for this project.
  • Parent item — when this item has a parent, the parent's custom values flow in.
  • This item (highest priority) — overrides everything else for this item.

Media directives — attach files to the post (substitute to empty in the body).

  • {{video}} — attach the item's primary video file.
  • {{thumbnail}} — attach the item's thumbnail.
  • {{image:cat}} — attach the image whose shortname is cat (set when uploading the image).
  • {{image:*}} — attach every image, in upload order.

AI blocks — sent to Claude to generate content. Variables inside an AI block are resolved before the prompt ships.

  • {{ai: prompt text}} — one Claude call using the default system prompt.
  • {{ai[system text]: prompt text}} — per-block system-prompt override (only this block uses it).
  • Nested AI blocks are evaluated leaves-first. The inner output is spliced into the outer prompt, then the outer call runs.
  • Sibling AI blocks at the same level are independent. Each is one Claude call.

Examples

  • {{ai: Write a punchy tweet about {{title}}}} {{url}}
  • Watch the full episode: {{episode_url!}} — clip below: {{video}}
  • {{ai[Be terse, lowercase only]: 3-word summary of {{description_short}}}}
  • {{ai: Tweet about "{{title}}" using this hashtag list: {{ai: Write 3 CamelCase hashtags for {{tags}}}}}}
  • Photos from today {{image:*}} — more on {{project_url}}

All renderings (preview, generate-posts, auto-actions, prompt-template bodies) go through the same engine — the POST /api/expand_text endpoint exposes it directly.

Slots

Each slot produces one social post. Built-in slots are seeded with the template and can be edited or disabled, but not deleted. Add extra slots to post a platform more than once (e.g. two LinkedIn accounts) or to extend the template to a new platform.
