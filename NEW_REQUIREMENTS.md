### The big changes

Right now everything is set up to support a single <everything>. A single youtube account, a single set of social media accounts.

We need to add some layers here.

We need to support:
- Multiple youtube channels - this will involve oauthing for different ones and storing things separately.
- Different social media accounts - again, more sets of credentials
- Easy use of the same social media account across multiple channels/projects

Within a given channel we also need to break down videos into separate tiers or tracts:
- video - a main content video  - 12 minutes or longer 
- segment - a segment of a main content video > 180 seconds and less than 12 minutes
- short - vertical video, duration matching a youtube short 50 seconds through 180 seconds
- hook - very short video -- less than 50 seconds

For each type (tract/tier) will have its own selection of templates as described above.

Proposed new structure adds a 'Project' layer
- Each project needs a name
- and of course a unique ID
- a project is associated with a SINGLE YouTube channel
- Each project has its own set of social media credentials
  - HOWEVER, we must support easy reuse across projects
  - Social media credentials must always list a username. Ideally we can get this from the service. If not, it can bee something the user enters when setting them up.
    - This allows us:
      - (1) to easily reuse them if we want to use the same social media account in multiple projects., and
      - (2) We can display it in the UI when composing posts, so the user is crystal clear on what account they're posting to.
- Project name will appear on dashboard

Proposed hierarchical structure:
- Home -- new top level that displays these 3 components:
  - Upcoming - shows upcoming scheduled things (newest first) - up to 7 lines
  - Recent - shows recent log messages across all projects. a maximum of 7 lines
  - Projects (new section to list the projects)
    - Add project button
    - Each project name shown in a card, with some stats, number of videos, # of scheduled events, whatever we can put that makes sense.
    - Tapping on project brings Dashboard
    - Dashboard
      - Like dashboard now. Title has Project Name / Dashboard - otherwise just like now.
        - Video detail - just like now but with the many changes as described above
    - Moderation - same as now
    - Templates - like now, but with many changes as described above
    - Project Settings
      - Includes youtube account setup, social media credentials (which must allow selection of an existing one), and pinned links
- General Settings
  - Anthropic API key
  - Anthropic model name
  - Background service
  - Comment check interval
  - Caption check interval

So the left side navigation will always have
- Home
- General Settings

But when a project is selected:
- Home
  - "Project name" Dashboard
  - Moderation
  - Templates
  - Project settings
- General settings

To best support multiple social media and youtube accounts, we should use OAUTH 2.0 as much as possible, provided that it gives us all of the functionality we want.
